# CallCenterOpt — Оптимизатор штата колл-центра

Курс: Теория игр | Итоговый проект

Авторы: Румянцев В.И. и Суханов Я.В.

## 1. Описание проекта

Python-приложение, моделирующее работу колл-центра как системы массового обслуживания ($M/M/c$) и подбирающее оптимальное количество операторов. Выбор стратегии найма формализуется как игра с природой в условиях риска (неопределённый поток звонков).

**Цель:** минимизация затрат на персонал при соблюдении строгого ограничения SLA ($P_{wait} \le 0.15$).

## 2. Математическая модель

Проект объединяет три математических аппарата:
* **СМО ($M/M/c$):** Пуассоновский входящий поток ($\lambda$), экспоненциальное время обслуживания ($\mu$), расчёт вероятности ожидания ($P_{wait}$) и длины очереди ($L_q$).
* **Игра с природой:** Сценарии нагрузки $\lambda \in \{10, 30, 60\}$ против стратегий найма $c \in \{2, 4, 6, 8, 10\}$. Применяются критерии Вальда, Гурвица ($\alpha=0.7$), Байеса и Сэвиджа.
* **Линейная оптимизация (ЛП):** Валидация базового сценария с использованием симплекс-метода (PuLP) для минимизации затрат.

## 3. Технологический стек

* **Ядро:** Python 3.12, NumPy, SciPy (математика СМО), PuLP (линейное программирование).
* **Инфраструктура:** Pydantic (строгая валидация конфигурации), PyYAML.
* **Визуализация:** Matplotlib.
* **Качество кода:** pytest (покрытие математического ядра 100%, игровых моделей 62%).

## 4. Структура проекта

* `config/` — конфигурационные файлы (`scenario.yaml`).
* `core/` — математическое и логическое ядро:
  * `queue_math.py` — формулы Эрланга C и расчёт метрик СМО.
  * `game_solver.py` — построение платёжной матрицы и игровые критерии.
  * `optimizer.py` — математический эталон на базе линейного программирования.
  * `config_schema.py` — Pydantic-схемы валидации.
* `tests/` — unit-тесты (`test_queue_math.py`, `test_game_solver.py`).
* `main.py` — точка входа и CLI-интерфейс.
* `visualize.py` — модуль построения тепловых карт и графиков SLA.

## 5. Быстрый старт и запуск

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   pip install -r requirements.txt
    ```

3. Запустите полный анализ с применением всех критериев и генерацией графиков:
    ```bash
    python main.py --all --plot
    ```



## 6. Результаты

По результатам выполнения сценария:

* Оптимальное число операторов по математическому эталону (ЛП под среднюю нагрузку) составляет **5 человек**.
* Оптимальное число операторов с учётом игровых рисков (по критериям Вальда, Гурвица, Байеса и Сэвиджа) составляет **6 человек**, что гарантирует соблюдение SLA во всех сценариях, кроме экстремального ($\lambda=60$).
